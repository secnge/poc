F5 PoC — Discord patch-gap delivery

Authorized security-research artifact. This page demonstrates the V8 13.8 asm.js heap-OOB bug class (V8 fix c0a41078) reaching Discord's renderer through an iframe surface. It is intended for HackerOne submission to hackerone.com/discord. Do not deploy against systems you do not own.

What happens when this URL is loaded inside Discord

  1. Discord fetches this page server-side for the embed preview.
  2. If Discord's embed flow iframes og:video with og:video:type=text/html (Hacktron's 2021 surface), iframe_trigger.html loads in the chat.
  3. Inside that iframe, the asm.js fragment compiles, calls m.f(1,2,3), and V8's JIT emits a heap-OOB write past the Int32Array's ArrayBuffer.
  4. The renderer process traps with STATUS_ACCESS_VIOLATION (0xC0000005).
  5. Discord's WebContents handler logs render-process-gone { reason: 'crashed', exitCode: -1073741819 } and reloads.
